Introduction

Git is a distributed version control system used to track changes in source code during software development. It helps teams collaborate efficiently, manage code history, and prevent loss of work. This documentation introduces Git basics and essential commands.

What is Git?

Git is a version control system that records changes to files over time. It allows multiple developers to work on a project simultaneously without interfering with each other. Git also keeps a detailed history of code changes, making it easier to revert to previous versions if needed.

Installing Git

After installation, use git --version to verify Git is installed.

Git Basics

After installing Git, you can start using it:

  • git init initializes a new Git repository.
  • git add stages files for commit.
  • git commit saves staged changes.
  • git status shows current state of working directory.
  • git log displays commit history.

Branching and Merging

Branches allow parallel development.

  • git branch <name> creates a new branch.
  • git checkout <name> switches to a branch.
  • git merge <name> merges another branch into your current one.

Branching helps isolate features or fixes before merging them into the main codebase.

Useful Commands

Some additional useful Git commands:

  • git clone clones a remote repository.
  • git pull updates your local repo from the remote.
  • git push uploads your changes to the remote.
  • git remote -v lists connected remotes.

These commands are essential for working with remote repositories like GitHub.

Conclusion

Git is a powerful tool that improves code collaboration and tracking. Understanding basic Git commands and workflows enables developers to work efficiently and reduce errors. Mastering Git is a valuable skill for any developer.
